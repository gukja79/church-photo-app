# Apps Script 백엔드 배포 가이드

이 문서는 처음 해보는 분도 따라할 수 있도록 단계별로 자세하게 작성했습니다.
순서대로 따라가시면 됩니다. 막히면 각 단계 아래의 "막혔을 때" 항목을 확인해주세요.

## 0. 사전 준비물

- Google 계정 (이 프로젝트는 `igodswill.org` 워크스페이스 계정으로 진행)
- 사진을 저장할 **공유 드라이브 폴더**의 ID:
  - 위치: `공유 드라이브 > 교회행정 > 서울고 관련 > 서울고 교실 사진`
  - **폴더 ID**: `1_iFeio5Z-knpa5GjeIY0-x9bvqJyS8tU`
  - (폴더 URL이 `https://drive.google.com/drive/folders/<ID>` 형식이면 마지막 부분이 ID)

> 💡 **중요**: Apps Script가 이 공유 드라이브 폴더에 파일을 만들려면, 배포 계정이 그 폴더의 **편집자 권한** 이상을 가지고 있어야 합니다. (지금 사용 중인 igodswill.org 계정이 이미 권한이 있으면 OK)

## 1. Apps Script 프로젝트 만들기

1. 브라우저에서 [https://script.google.com](https://script.google.com) 접속
2. 좌측 상단 **"+ 새 프로젝트"** 클릭
3. 상단의 **"제목 없는 프로젝트"** 부분을 클릭하여 이름을 `교회 교실 사진 백엔드`로 변경

### 1-1. 코드 붙여넣기

1. 좌측 파일 트리에서 기본으로 생긴 `Code.gs` 클릭
2. 편집창의 모든 내용을 지우고, 이 저장소의 [`apps-script/Code.gs`](./Code.gs) 파일 전체를 복사하여 붙여넣기
3. 좌측 상단의 💾(저장) 아이콘 또는 `Cmd+S` / `Ctrl+S`로 저장

### 1-2. (선택) 프로젝트 매니페스트 파일 동기화

이 단계는 시간대 자동 설정을 정확하게 해줍니다. 안 해도 동작에는 문제 없지만, 시간 관련 버그를 줄여줍니다.

1. 좌측 톱니바퀴 모양의 **"프로젝트 설정"** 클릭
2. **"appsscript.json' 매니페스트 파일을 편집기에 표시"** 체크박스 켜기
3. 좌측 파일 트리로 돌아오면 `appsscript.json`이 보임
4. 그 파일 내용을 [`apps-script/appsscript.json`](./appsscript.json) 내용으로 교체하고 저장

## 2. 스크립트 속성 설정 (PARENT_FOLDER_ID)

이 부분이 빠지면 업로드 시 에러가 납니다. 꼭 해주세요.

1. 좌측의 톱니바퀴 모양 **"프로젝트 설정"** 클릭
2. 페이지 아래쪽 **"스크립트 속성"** 섹션의 **"스크립트 속성 추가"** 버튼 클릭
3. 다음과 같이 입력:
   - **속성**: `PARENT_FOLDER_ID`
   - **값**: `1_iFeio5Z-knpa5GjeIY0-x9bvqJyS8tU`
4. **"스크립트 속성 저장"** 클릭

> 💡 **PIN 설정** (3단계 추가됨)
>
> PIN을 설정하지 않으면 누구나 앱을 쓸 수 있습니다. 운영 시작 전에 반드시 설정하세요.
>
> 1. 같은 화면(스크립트 속성)에서 **"스크립트 속성 추가"** 한 번 더 클릭
> 2. **속성**: `PIN`
> 3. **값**: 4자리 숫자 (예: `2580`)
> 4. **"스크립트 속성 저장"** 클릭
>
> PIN을 처음 설정하거나 바꾸면 **모든 사용자가 다음 접속 시 새 PIN으로 다시 로그인**해야 합니다.
> (백엔드가 토큰의 PIN 지문을 검증하기 때문에 자동 무효화됩니다. 별도 작업 불필요.)
>
> **`SECRET` 속성**은 첫 토큰 발급 시 자동 생성됩니다. 직접 만들 필요 없습니다.
> - 이 키가 노출되면 토큰 위조가 가능해지므로, 외부에 공유하지 마세요.
> - 의심스러우면 `SECRET` 값을 삭제하면 자동 재생성되며 모든 기존 토큰이 무효화됩니다.

## 3. 권한 부여 + 동작 테스트

업로드 코드는 Drive에 폴더와 파일을 만들 권한이 필요합니다. 처음 한 번만 권한 승인을 받으면 됩니다.

1. 편집기로 돌아와 좌측의 함수 선택 드롭다운에서 **`testSetup`** 선택
2. **"실행"** 버튼 클릭
3. 처음에는 권한 승인 창이 뜸:
   - **"권한 검토"** → 사용할 Google 계정 선택
   - "이 앱은 Google에서 인증되지 않았습니다" 경고가 뜨면:
     - **"고급"** 클릭 → **"교회 교실 사진 백엔드(안전하지 않음)으로 이동"** 클릭
     - (이건 본인이 만든 코드이므로 안전합니다)
   - 요청 권한들 확인 후 **"허용"**
4. 다시 한 번 **"실행"** 클릭 → 하단에 **실행 로그** 탭이 뜸
5. 다음과 비슷한 출력이 보이면 성공:
   ```
   부모 폴더 이름: 서울고 교실 사진
   부모 폴더 URL: https://drive.google.com/...
   PIN 설정 여부: 아니오 (개발 모드)
   서버 시간(KST): 2026-04-29T...
   ```

### 막혔을 때

- **"PARENT_FOLDER_ID가 설정되지 않았습니다"** → 2번 단계의 스크립트 속성을 다시 확인
- **"파일을 찾을 수 없음"** 비슷한 오류 → 폴더 ID가 정확한지, 그 공유 드라이브에 본인 계정이 편집자 권한인지 확인

## 4. 웹앱으로 배포

이제 외부(아이폰 사파리, 데스크톱 등)에서 호출 가능한 URL을 만듭니다.

1. 편집기 우측 상단의 **"배포"** 드롭다운 → **"새 배포"** 클릭
2. 배포 다이얼로그 좌측의 톱니바퀴 아이콘 → **"웹 앱"** 선택
3. 다음과 같이 설정:
   - **설명**: `v1` (선택)
   - **다음 사용자 인증 정보로 실행**: `나` (배포한 본인 계정)
   - **액세스 권한이 있는 사용자**: **`모든 사용자`** ← 중요
4. **"배포"** 클릭
5. 배포가 완료되면 **"웹 앱 URL"**이 표시됨. 형식은:
   ```
   https://script.google.com/macros/s/AKfycby......./exec
   ```
6. 이 URL을 복사하여 어딘가에 임시 저장 (5번 단계에서 프론트에 붙여넣음)

### 코드를 수정해서 다시 배포하고 싶을 때

- **"배포"** → **"배포 관리"** → 기존 배포 옆 ✏️(연필) 아이콘 → **"새 버전"** 선택 → **"배포"**
- 이렇게 하면 **URL이 바뀌지 않습니다**. (새 배포로 만들면 URL이 새로 생성됨)

## 5. 프론트엔드에 API URL 연결

방법 두 가지 중 편한 쪽으로 선택하세요.

### 방법 A: 코드에 직접 박기 (배포 후 권장)

`js/api.js` 파일을 열고 다음 줄을 찾아 URL을 채워 넣습니다.

```javascript
const API_URL = '';  // ← 여기
```

```javascript
const API_URL = 'https://script.google.com/macros/s/AKfy.../exec';
```

저장하고 GitHub에 push (5단계에서 GitHub Pages 배포할 때).

### 방법 B: 브라우저에서 임시로 덮어쓰기 (개발/테스트용)

브라우저에서 앱 페이지를 연 다음 개발자 도구(콘솔)에서 한 번만 실행:

```javascript
localStorage.setItem('API_URL_OVERRIDE', 'https://script.google.com/macros/s/AKfy.../exec')
```

이후 새로고침하면 그 URL을 사용합니다. 사파리(아이폰)에서도 같은 도메인이면 유지됩니다.

## 6. 동작 확인

1. 로컬 서버 띄우기:
   ```bash
   cd church-photo-app
   python3 -m http.server 8080 --bind 0.0.0.0
   ```
2. 브라우저에서 `http://localhost:8080` (또는 같은 와이파이의 아이폰에서 `http://<맥IP>:8080`)
3. 장소 선택 → 모임 전 → 사진 찍기 → 업로드
4. 업로드 후 공유 드라이브의 부모 폴더에 다음 구조로 생성되는지 확인:
   ```
   서울고 교실 사진/
   └── 2026-04-29/
       └── 2층/
           └── 1-14/
               └── 1_전_2층_1-14_140523_001.jpg
   ```
5. 홈 화면으로 돌아왔을 때 그 장소 카드의 점이 파랗게 (또는 초록으로) 변하고 "Before 1장 / After 0장"이 표시되면 성공

### 막혔을 때

- **"API_URL이 설정되지 않았습니다"** → 5번 단계 다시 확인
- **CORS 오류** → 5번에서 URL을 정확히 복사했는지, 4번에서 "액세스 권한"이 "모든 사용자"인지 확인
- **"PIN 인증 실패"** → 2번 단계에서 `PIN` 속성을 추가했는데 프론트에서 PIN을 안 넣고 있는 상태. 일단 `PIN` 속성을 삭제하거나, 브라우저 콘솔에서 `localStorage.setItem('PIN', '여기에 PIN')` 실행. (PIN은 3단계에서 본격 작업)
- **403 / "권한이 없습니다"** → 부모 폴더에 Apps Script 실행 계정이 편집자 이상 권한이 있는지 확인
- **업로드는 되는데 "Before 0장"으로 보임** → 파일명이 `1_전_...` 또는 `2_후_...`로 시작해야 카운팅됨. Drive에서 파일명을 확인해보세요.

## 7. PIN 변경 / 장소 추가 (추후 운영)

- **PIN 변경**:
  1. Apps Script 편집기 → 좌측 **프로젝트 설정**(톱니바퀴) → 스크립트 속성에서 `PIN` 값 수정 → 저장
  2. 재배포 **불필요**. 변경 즉시 반영됨.
  3. **자동 효과**: 모든 기존 사용자의 토큰이 자동 무효화 → 다음 접속 시 새 PIN으로 다시 로그인 필요.
  4. 새 PIN을 자원봉사자분들께 안내해주세요.
- **PIN을 잊었을 때**:
  - 위와 동일하게 `PIN` 속성 값을 새로 정해서 덮어쓰면 됩니다.
- **모든 토큰을 한꺼번에 강제 만료시키고 싶을 때**:
  - 스크립트 속성에서 `SECRET` 값을 삭제하면 다음 토큰 발급 시 자동 재생성되고, 기존 토큰은 모두 검증 실패.
- **PARENT_FOLDER_ID 변경**: 같은 위치에서 값만 바꿈 (재배포 불필요)
- **장소 추가/수정**: `js/locations.json` 편집 후 GitHub에 push (백엔드 변경 없음)
- **서버 코드 변경**: 4번의 "코드를 수정해서 다시 배포..." 참고

## 부록 A: API 엔드포인트 명세

### POST `/exec` (action: upload)

**요청 (Content-Type: text/plain;charset=utf-8 권장 - CORS 프리플라이트 회피)**
```json
{
  "action": "upload",
  "pin": "1234",
  "date": "2026-04-29",
  "floor": "2층",
  "locationId": "1-14",
  "locationName": "1-14",
  "phase": "before",
  "filename": "1_전_2층_1-14_140523_001.jpg",
  "mimeType": "image/jpeg",
  "data": "base64로_인코딩된_이미지"
}
```

**응답**
```json
{
  "success": true,
  "fileId": "1abc...",
  "fileUrl": "https://drive.google.com/file/d/.../view",
  "filename": "1_전_2층_1-14_140523_001.jpg",
  "folderPath": "2026-04-29/2층/1-14",
  "uploadedAt": "2026-04-29T14:05:23+09:00"
}
```

### GET `/exec?action=status&date=2026-04-29`

**응답**
```json
{
  "success": true,
  "date": "2026-04-29",
  "locations": {
    "1-14": { "before": 2, "after": 0 },
    "여자화장실": { "before": 0, "after": 1 }
  }
}
```

키는 폴더 이름(=장소 표시명)입니다. 프론트는 `location.name`으로 매칭합니다.

### POST `/exec` (action: auth)

**요청**
```json
{ "action": "auth", "pin": "2580" }
```

**응답 (성공)**
```json
{
  "success": true,
  "token": "eyJleHAiOjE3..ZX0.AbCdEf...",
  "expiresAt": 1759999999999
}
```

**응답 (실패)**
```json
{ "success": false, "error": "PIN이 틀렸습니다" }
```

**토큰 형식**: `base64url(payload).base64url(HMAC-SHA256 sig)`
- `payload = { exp, iat, v }` — `v`는 PIN의 짧은 해시
- 서명 키는 자동 생성되는 `SECRET`
- PIN이 바뀌면 `v` 불일치로 자동 무효화

이후 `upload`, `status` 요청에는 `token` 필드를 함께 보내야 합니다. 없으면:
```json
{ "success": false, "error": "인증이 필요합니다", "code": "AUTH_REQUIRED" }
```

### GET `/exec?action=ping`

서버 동작 확인용. `{ "success": true, "ping": "pong", "time": "..." }` 반환.

## 부록 B: 향후 확장 메모 (6단계 관리자 페이지용)

지금 백엔드는 다음 액션을 받을 준비가 되어 있습니다 (확장 시 같은 패턴):
- `upload`, `auth`, `status`

6단계에서 추가할 수 있는 액션 예시:
- `listLocations` / `addLocation` / `updateLocation` / `deleteLocation` (장소 관리)
- `dailySummary` (전체 통계)
- `bulkDownload` (사진 일괄 다운로드)

`doPost`/`doGet`의 `action` 분기에 핸들러만 추가하면 됩니다.
